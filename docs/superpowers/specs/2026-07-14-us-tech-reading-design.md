# 美股科技阅读课 — 设计规格

**日期：** 2026-07-14  
**仓库名：** `us-tech-reading`（独立站；GitHub：`caoyuchun2003/us-tech-reading` 待创建）  
**状态：** 待用户审阅后进入实现计划

---

## 1. 产品共识

| 项 | 决定 |
|---|---|
| 品类 | 个人教育站（非资讯 / 行情工具 / 社区） |
| 定位一句话 | 给国内新手的美股科技公司阅读课 |
| 流派 | 质量复合价值投资（好生意 + 护城河 + 合理估值）；非烟蒂、非短线 |
| 市场 | 纳斯达克 / 美股科技 |
| 主键用户 | 国内新手；进阶仅为附属内容 |
| 获客（首期） | 搜索 + 长文 |
| 交付 | 8 课序列课；模板化案例作作业 |
| 主案例 | Apple；对照案例暂无 |
| 变现 | 前 3 课免费 + 后 5 课整包；首发 ¥99 |
| 收款 | 微信 / 支付宝个人收款码；人工发兑换码 |
| 战绩披露 | 弱披露（多年实盘 + 回撤真实感）；首页不写收益倍数 |
| 节奏 | 每周约 8–12 小时；约 12 周内冲第一笔付费 |
| 失败线 | 几乎无人读完免费前 3 课 → 先改内容/获客，不堆功能 |

### 1.1 课纲（已整单接受）

1. 质量价值是什么（和博弈 / 听消息的区别）  
2. 能力圈与美股科技：你到底看懂什么  
3. 怎么读懂生意（用户、收入结构、单位经济）— Apple 切入  
4. 护城河：网络效应、转换成本、规模经济（科技例子 / Apple）  
5. 财报抓手：收入质量、毛利、经营利润、FCF  
6. 估值常识：倍数在说什么、何时失效（无目标价）  
7. 风险清单：竞争、监管、估值压缩、个人行为  
8. 作业：按同一模板拆解 Apple（教学案例），产出「一页纸」

### 1.2 合规红线

- 可使用具体公司作**教学案例**  
- **禁止**：买入/卖出/加仓指令、目标价、仓位比例建议  
- 操作类词汇仅用于解释概念，不对具体标的下指令  
- 全站页脚短免责；`/disclaimer` 长文；付费页标明「教育产品」  
- 关于页可弱披露实盘经历，不展示未审计收益倍数或曲线广告

## 2. 目标与非目标

### 2.1 MVP 成功标准

- 访客能从首页一键进入第 1 课并完成免费前 3 课阅读  
- 付费用户凭兑换码解锁第 4–8 课  
- 全站免责与措辞符合红线  
- 12 周内能完成人工收款 → 发码 → 解锁闭环  

### 2.2 明确不做（MVP）

- 实时行情、看板、选股器  
- 社区、评论、跟单  
- 完整用户账号体系 / OAuth  
- 支付自动回调与订单系统  
- AI 荐股或自动生成买卖建议  
- CMS 后台

## 3. 技术架构

**选型：** VitePress（静态文档站，Markdown 成课）

**理由：** 匹配「序列课 + 侧栏课纲」；作者已有 VitePress 使用经验（`learn` 仓库）；支付与账号保持人工/轻量，把时间留给课文。

### 3.1 页面地图

| 路径 | 职责 |
|---|---|
| `/` | 课表优先落地页：定位 → CTA「开始第 1 课」→ 8 课清单（标免费/付费）→ 免责摘要 |
| `/course/01` … `/course/08` | 课文 |
| `/pay` | ¥99 说明、收款方式、如何联系获取兑换码 |
| `/unlock` | 输入兑换码；成功则本地持久化访问权限 |
| `/about` | 作者弱披露 + 指向完整免责 |
| `/disclaimer` | 完整非投资建议声明 |

首页信息架构：**课表优先**（非杂志流、非双栏仪表盘）。

### 3.2 内容模型

- 课文路径建议：`course/01-quality-value.md` … 或 `course/01.md` … `08.md`  
- Frontmatter 至少包含：`title`、`order`、`free`（boolean）  
- 第 8 课附「一页纸」作业模板（可同页或 `course/worksheet.md`）  
- MVP 可用手写首页课表列表；不强制从 glob 自动生成  

### 3.3 付费闸门

```
访客 → 读 01–03（free: true）
     → 访问 04–08 → 检查 localStorage token
         → 有效：渲染正文
         → 无效：遮罩 + 链到 /pay 与 /unlock
/pay → 人工转账 → 运营者发兑换码
/unlock → 校验码 → 写入 localStorage → 跳转上次或第 4 课
```

**兑换码存储：**

- 合法码不进入公开 Git 明文  
- 使用构建时环境变量（如 `VITE_UNLOCK_CODES` 哈希列表）或仅部署环境注入  
- 客户端校验为 MVP 权宜：防君子不防黑客；说明书中接受「可被绕过」，真正防盗靠更新码与内容迭代，而非 DRM  

**实现要点（实现计划阶段细化）：**

- VitePress 自定义 layout 或 Vue 组件包裹付费章正文  
- `unlock` 页表单；校验对「盐 + 码」的哈希或与构建注入列表比对  
- Token 形如签过名的简单字符串或 `unlocked=1` + 码指纹，存 `localStorage`  

### 3.4 部署（默认）

- GitHub 仓库 + 静态托管（GitHub Pages 或 Cloudflare Pages / Vercel 静态）  
- 兑换码仅通过托管平台 Environment Variables 注入  
- 具体宿主在实现计划中选定一个，默认 **Cloudflare Pages 或 GitHub Pages**（零/低成本）

## 4. 视觉设计

**方向：墨蓝书房（已选定）**

| Token | 值 |
|---|---|
| 背景 | `#F5F7FA` |
| 主墨 / 链接 / CTA | `#1B3A5F` |
| 正文 | `#142033` |
| 次要文字 | `#5A6578` |
| 标题字体 | Noto Serif SC（或等价衬线） |
| 正文字体 | Noto Sans SC |
| 圆角 | ~2px，克制 |
| 禁止 | 紫霓虹、厚重多层阴影、荐股 ticker、圆角胶囊堆、假装「交易终端」的深色默认皮肤 |

首页第一个视口只保留：品牌/课名、一句定位、一个主 CTA、课表入口信号；不做统计条或促销贴片。

---

## 5. 文案与内容约束

- CTA 用「开始第 1 课 / 继续学习」，不用「立刻跟单」「稳赚」等  
- 案例叙述用过去时或结构化教学语气：「如何评价护城河」「财报里看什么」  
- Apple 材料基于公开信息与教学简化，标明数据截止口径（实现时在课内统一）  
- `/pay` 明确：购买的是**教育内容访问权**，不是投顾服务  

---

## 6. 度量（人工即可）

首期不接复杂分析，作者可手工看：

- 免费第 3 课是否有人读完（反馈 / 私信 / 你自建极简计数可选）  
- 兑换码发放数量 ≈ 付费单数  
- 失败判定：免费前 3 课几乎无人完读 → 停功能开发，改课与获客  

---

## 7. 风险与接受

| 风险 | 处理 |
|---|---|
| 客户端解锁可被绕过 | 接受；码可轮换；不宣传「防盗版」 |
| 个人收款合规与纠纷 | 小额教育内容；保留转账记录；不作收益承诺 |
| 「价值 × 科技股」被误解为荐股 | 免责 + 措辞审查清单（发布前自检） |
| 内容写不完 | 先上架 01–03 + pay/unlock 壳；04–08 可陆续放开 |

---

## 8. 里程碑（与共识对齐）

1. **脚手架：** VitePress + 主题变量 + 页面壳 + 免责  
2. **闸门：** pay / unlock + 付费章锁定  
3. **内容：** 写完并发布 01–03；课表与 CTA 可用  
4. **内容：** 04–08 + Apple 一页纸模板  
5. **获客：** 1～2 篇导向长文 / README；收集第一笔 ¥99
