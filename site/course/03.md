# 03 读懂生意

从本课起，Apple 会作为贯穿教学案例。目标不是「证明 Apple 该如何交易」，而是练习一套可迁移到其他美股科技公司的问题清单：用户是谁、钱怎么进来、单位层面是否讲得通。

> 教育材料，非投资建议。下文涉及的业务结构与数字，均为公开信息的教学简化，口径会随财报更新而变化；请勿据此形成买卖或目标价判断。

## 读懂生意的最低四问

打开任何科技公司材料前，先写这四问：

1. **用户**：谁在付钱？谁在使用？付费者与使用者是否同一人？
2. **工作**：产品完成了什么「工作」（Jobs to be done）？替代方案是什么？
3. **收入结构**：钱主要来自哪几块？哪一块增长、哪一块利润贡献更大？
4. **单位经济**：多服务一个用户，或公司每多出货一台设备，收入、成本、后续关联收入如何变化？

四问答不清，后面的护城河与估值都会漂。宁可在本课多花一小时把四问写疼，也不要急着跳到倍数讨论。

## 用户：Apple 的教学切入

用教学语言（非完整市场画像）看 Apple，可以这样拆：

- **个人消费者**：购买 iPhone、Mac、手表、耳机等硬件，并可能在系统内支付 App、媒体、云存储等服务。
- **家庭与多设备用户**：多台设备共用账户与服务，提高「离开成本」的感知。
- **开发者与内容方**：不是典型「买手机的人」，却通过 App Store 等机制参与价值分配——理解这类参与者，有助于读懂服务收入。
- **企业与教育场景**：存在，但对初学者可先知道「有」，再查分部披露；不要一开始就用传闻填满空白。

练习重点：把「我用过 iPhone」升级为「我知道付钱与使用可能不是同一步」。例如家长买设备、孩子用应用；公司采购设备、员工使用——这会影响你如何理解留存与切换。

## 收入结构：先分块，再谈叙事

阅读美股科技，最常见的进步是：**停止只用一个故事概括全部收入**。对 Apple 这类公司，公开财报通常会拆出硬件与服务等类别（具体科目名称以最新 10-K / 10-Q 为准）。教学上可先用如下骨架理解：

| 教学分块 | 你要问什么 |
|---|---|
| 旗舰硬件（如 iPhone） | 出货/换机周期、价格带、地区结构是否敏感？ |
| 其他硬件 | 是补充入口还是独立利润池？与旗舰如何协同？ |
| 服务 | 订阅、佣金、云、支付相关等，哪类更依赖装机量？ |
| 地区 | 增长或风险是否集中在少数市场？ |

重要纪律：**分块是为了找依赖关系**，不是为了 invent 精确到小数点的预测。你要训练的是：「如果换机放缓，服务是否仍能跟装机基数相连？」「服务毛利特征与硬件是否不同？」

当公开材料给出各类收入占比或同比时，把数字当作「结构快照」，并注明你记录的报告期。教学简化不等于忽略时效。

## 单位经济：不必会建模，但要会提问

「单位经济」听起来像金融工程，初学者只需掌握问题级定义：

- 公司每多出货**一台**主力设备，立刻确认的收入与成本大概如何思考？（硬件制造、渠道、保修等——用定性即可）
- 这台设备进入生态后，是否带来**后续服务收入**？服务是一次性还是经常性？
- 服务增量主要来自更多用户、更高 ARPU，还是更高使用强度？
- 哪些成本随用户量近似线性上升？哪些更像平台固定投入？

对 Apple 教学案例，一条常见线索是：硬件创造「装机入口」，服务与生态提高「使用深度与付费面」。这句话是分析假说，需要用财报分拆与产品事实去检验，而不是当作口号背诵。

你还不需要做完整 DCF。你需要做到：看到「营收增长」时，能追问增长来自价格、用量还是新品类；看到「利润率变化」时，能追问是结构迁移（例如服务占比）还是费用波动。

## 把生意写成可复盘的一段话

学完本课，试着把 Apple（教学案例）压成 8–12 行笔记，而不是小作文：

1. 核心用户与付费关系（两句）
2. 收入三大块及你盯的依赖（三句）
3. 单位层面你最在意的闭环（两句）
4. 目前公开材料里你仍读不懂的点（两三句，允许空白）

这段话会在第 4–8 课不断被护城河、财务抓手、估值含义与风险清单「加压」。现在写得粗糙没关系；没有这段话，后面只能继续复述标题。

## 常见初学者陷阱（读生意时）

- **用产品评价代替生意评价**：「影像很好」不等于收入结构健康。
- **用单一爆款解释全部增长**：忽略地区、价位与服务拉动。
- **把「我愿意付」外推成「所有人单位经济」**：个人体验样本极偏。
- **追精确预测而忽视依赖关系**：先写清「若 A 变慢，B 如何」，再谈精确度。

对照检查：若你的笔记里形容词远多于依赖箭头，就还没读懂生意。

## 本课小结

- 读懂生意 = 用户、工作、收入结构、单位经济四问闭环。
- Apple 用作贯穿案例，因其公开分拆与大众可感，便于练习；不是交易暗示。
- 数字请标记报告期与「教学简化」属性；后续课会继续用同一案例加深，但不引入买卖指令。
- 先建立可复盘的依赖描述，再考虑更细的财务与估值工具。
